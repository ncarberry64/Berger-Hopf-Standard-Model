# Artifact-backed claim policy

Allowed descriptions are limited to local artifact discovery, provenance-aware
loading, and explicit separation among internal outputs, interface defaults,
calibration inputs, reference comparisons, and missing theorem callables.
Artifact presence alone does not promote a theorem blocker.

The adapters do not establish empirical validation, independently predict a
calibration anchor, use PDG values as derivation inputs, infer missing values,
or promote interface placeholders. `X_ch`, the physical neutrino basis/scale,
and standalone CP `O_int` stay open until explicit theorem artifacts and
callables exist.

Artifact-backed outputs are local BHSM outputs with provenance, not empirical validation claims.

Interface default formulas remain interface defaults unless a theorem-backed artifact or callable replaces them.

Reference values, including PDG values, are comparison inputs only and are never BHSM derivation inputs.

Missing artifacts are reported as missing, not inferred.

Theorem blockers remain blockers unless explicit artifact-backed theorem support is present.
