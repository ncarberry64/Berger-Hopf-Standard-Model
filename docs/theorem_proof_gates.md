# Theorem proof gates

The closure framework evaluates seventeen mandatory gates:

1. formal theorem statement;
2. domain and codomain;
3. operator/action term;
4. callable implementation;
5. input/output schema;
6. artifact source;
7. provenance chain;
8. no empirical derivation inputs;
9. no reference or PDG derivation inputs;
10. consistency checks;
11. registry update proposal;
12. claim policy;
13. blocker artifact;
14. gauge/sector admissibility;
15. dimensional policy;
16. calibration policy;
17. documented failure modes.

`CLOSED_DERIVED_ACTION_LEVEL` is available only when every required gate
passes. An absent gate fails closed.

A partial localization is not a production prediction.
