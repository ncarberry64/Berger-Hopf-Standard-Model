# Python prediction claim policy

The registry strengthens usability by preserving categorical boundaries.

If a particle mass is used to calibrate the geometric-to-physical unit scale, that same particle cannot be counted as an independent prediction in that run.

The electron-neutrino comparison is treated as an upper-limit comparison unless a vetted central experimental mass reference is explicitly supplied.

Open-theorem entries are registry blockers, not production-ready predictions.

Runtime-disabled software entries require live external validation before readiness can be claimed.

Allowed descriptions include computational registry, calibration-aware model
output, frozen internal artifact, upper-limit comparison, open theorem blocker,
and runtime-disabled gate. The registry does not establish empirical
validation, official CERN integration, a complete 4D Lagrangian, or validated
FeynRules/UFO/MadGraph readiness.

The consolidated release claim table is
`docs/bhsm_v1_2_0_prediction_claim_status.md`.
