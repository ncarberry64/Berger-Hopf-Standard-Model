# Theorem closure claim policy

Executable closure attempts may report source artifacts, verified components,
failed proof gates, and exact missing objects. They may not infer a theorem from
prose, plots, notebooks, reference values, or calibration anchors.

`X_ch` is not production-ready without its action theorem. `K_nu` is not a
physical neutrino mass matrix without its physical map. A CP holonomy phase is
not a standalone interaction without `O_int`. Runtime software gates stay
disabled until their named external tools pass live validation.

Theorem closure requires executable artifact-backed support; narrative plausibility is not enough.

Reference values, including PDG values, are comparison inputs only and are never theorem inputs.

A partial localization is not a production prediction.

Runtime-disabled software gates remain disabled until live external validation passes.

The neutral operator kernel is not promoted to a physical neutrino mass matrix without a physical basis, dimensional scale, and Dirac/Majorana convention.
