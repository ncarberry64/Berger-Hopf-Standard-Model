# BHSM Claim Boundaries

## Allowed

- BHSM has frozen internal prediction artifacts.
- BHSM has an offline Python computational interface.
- BHSM has a prediction registry and CLI.
- BHSM has artifact-backed adapters with provenance for CKM, PMNS, CP phase,
  boundary constants, and mass ratios where local artifacts support them.
- BHSM has executable theorem-closure machinery.
- BHSM has a CP `O_int` symbolic field/action candidate, but not an action-level
  production theorem.

## Not Supported

Do not claim that:

- BHSM is empirically validated.
- BHSM is officially CERN integrated.
- BHSM has complete 4D Lagrangian export.
- BHSM has validated FeynRules/UFO/MadGraph readiness.
- W is an independent prediction in a run where W calibrates the unit scale.
- Electron-neutrino mass is centrally measured or centrally validated.
- CP holonomy alone proves a standalone production interaction.
- A symbolic CP `O_int` candidate is production-ready.

## Policy

Status follows evidence. Artifact-backed means a local source and provenance
path exist. Candidate means an explicit construction exists with named proof
gaps. Open means the missing object is identified. Runtime-gated means a live
external validation step remains. These labels do not substitute for empirical
comparison or theorem proof.
